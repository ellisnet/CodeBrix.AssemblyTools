CodeBrix.AssemblyTools is a .NET-10 port of Mono.Cecil 0.11.6 --- a fully
managed library for reading, writing, and rewriting .NET assemblies, IL,
and debug symbols. Because this project is a port, it incorporates source
code from Mono.Cecil (and a small embedded Microsoft CCI PDB reader /
writer that Mono.Cecil in turn carries). The details of each upstream
component --- its authors, source location, license, and which files in
this repository were derived from it --- are given below.

================================================================================

1. Mono.Cecil
--------------------------------------------------------------------------------

Project:   Mono.Cecil
Source:    https://github.com/jbevain/cecil
Version:   0.11.6 (git tag 0.11.6)
License:   MIT
Copyright: Copyright (c) 2008 - 2015 Jb Evain
           Copyright (c) 2008 - 2011 Novell, Inc.

CodeBrix.AssemblyTools incorporates essentially all of the Mono.Cecil 0.11.6
source tree, re-namespaced from "Mono.*" / "Mono.Cecil.*" into
"CodeBrix.AssemblyTools.*". Every ported source file carries a
"//was previously: <upstream namespace>;" comment on its `namespace` line so
the upstream origin of each file is trivially recoverable. The upstream
top-of-file copyright headers are preserved verbatim on every ported file.

This port was originally taken from the Mono.Cecil 0.11.5 tag and
subsequently brought up to 0.11.6 by cherry-picking every source-level
commit between the two tags:

  * 04286ac  Add public API for instruction cloning (Instruction.GetPrototype)
  * 56d4409  Fix privatescope method resolution bug
  * a840bdb  Update upstream test harness to .NET 8.0 (not applied --- our
             tests already target net10.0)
  * ba9c6c7  Bump .NET-Framework reference assemblies to v1.0.3 (not applied
             --- our port does not target .NET Framework)
  * fec4ee9  Import reflection modifiers (preserve in/ref-readonly/required/
             init-only modreqs on System.Reflection import)
  * dff01d9  Use FIPS-compliant SHA1 for strong-name signing
             (in our net10.0 port, implemented as `SHA1.Create()`)
  * 1da2145  Add new flag for ByRefLike generic-parameter constraint
             (C# 13 `where T : allows ref struct`)
  * 8e1ae7b  Resolve `.dll` before `.exe` in BaseAssemblyResolver
             (fixes BadImageFormatException on modern .NET apphost layouts)
  * 608fac6  Fix WriteCompressedInt32 logic by mirroring the
             System.Reflection.Metadata reference implementation
             (negative-value encoding corruption on 0.11.5)
  * 50292e7  Fix: SymWriter COM object is not released on exception
             (intermittent file-locking when writing native PDBs)
  * a0f61f9  Preserve custom debug information on types
             (ICustomDebugInformationProvider is now implemented by
             TypeDefinition; type-level PDB metadata survives round-trips)
  * ed62c02  Upstream README tidy (not applied)
  * 5de7d8c  Upstream version bump (not applied)

Modifications made during the port:
  * Namespace rename: "Mono"           -> "CodeBrix.AssemblyTools.Internal"
                      "Mono.Cecil"     -> "CodeBrix.AssemblyTools"
                      "Mono.Cecil.Cil" -> "CodeBrix.AssemblyTools.Cil"
                      "Mono.Cecil.Metadata" -> "CodeBrix.AssemblyTools.Metadata"
                      "Mono.Cecil.PE"  -> "CodeBrix.AssemblyTools.PE"
                      "Mono.Collections.Generic"
                                       -> "CodeBrix.AssemblyTools.Collections.Generic"
                      "Mono.Security.Cryptography"
                                       -> "CodeBrix.AssemblyTools.Security.Cryptography"
                      "Mono.Cecil.Rocks" -> "CodeBrix.AssemblyTools.Rocks"
                      "Mono.Cecil.Pdb" -> "CodeBrix.AssemblyTools.Pdb"
                      "Mono.Cecil.Mdb" -> "CodeBrix.AssemblyTools.Mdb"
                      "Mono.CompilerServices.SymbolWriter"
                                       -> "CodeBrix.AssemblyTools.Mdb.SymbolWriter"
  * Target framework narrowed from {netstandard2.0; net40} down to {net10.0}
    only. Conditional-compile branches selecting non-.NET-10 code paths
    (e.g. `#if NET_CORE`, `#if !NET_4_0`) were collapsed to keep only the
    modern path and delete the obsolete branches entirely.
  * Block-scoped namespace declarations (`namespace X { ... }`) converted
    to file-scoped declarations (`namespace X;`) throughout.
  * Library projects (Mono.Cecil, Mono.Cecil.Rocks, Mono.Cecil.Pdb,
    Mono.Cecil.Mdb) merged into a single assembly: CodeBrix.AssemblyTools.
  * The four upstream NUnit test projects (core, Rocks, Pdb, Mdb) merged
    into a single xUnit v3 test project: CodeBrix.AssemblyTools.Tests.
    Assertion style converted from NUnit `Assert.AreEqual` / `Assert.IsTrue`
    to a mix of xUnit `Assert.Equal` / `Assert.True` and SilverAssertions
    fluent `.Should().Be(...)` / `.Should().BeTrue()`.

3. Binary test fixtures (Test/Resources/**)
--------------------------------------------------------------------------------

Source:    https://github.com/jbevain/cecil at tag 0.11.6
           (paths Test/Resources/, rocks/Test/Resources/,
           symbols/pdb/Test/Resources/, symbols/mdb/Test/Resources/)
License:   MIT (shipped under the Mono.Cecil 0.11.6 LICENSE.txt --- see
           Section 1 above for the full text)

CodeBrix.AssemblyTools's test project includes Mono.Cecil's pre-built binary
test fixtures (small .dll / .exe / .pdb / .mdb / .cs / .il input files) used
as test inputs. These are copied verbatim and distributed under Mono.Cecil's
MIT license. If any specific fixture turns out to carry a more restrictive
upstream license, the test that uses it will be flagged and either replaced
or excluded in a follow-up pass.
